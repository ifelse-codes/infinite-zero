## Step-by-Step Instructions

### 1. Pre-flight Check
- Verify Docker Desktop is running
- Verify pnpm is installed (`pnpm --version` should be ~8.15.1)
- Verify Node.js version compatibility

### 2. Environment Setup
```bash
cd c:/playground/try-proliferate/repo
./scripts/setup-env.sh
```
- If `setup-env.sh` fails on Windows, manually copy `.env.example` → `.env` and generate secrets
- Document any Windows-specific issues in `.claude/debugger/KNOWN_ISSUES.md`

### 3. Configure .env
Minimum required keys to get running:
- `DATABASE_URL` (auto-generated by setup script)
- `NEXTAUTH_SECRET` / auth secrets (auto-generated)
- `GITHUB_APP_ID`, `GITHUB_APP_PRIVATE_KEY`, `GITHUB_APP_CLIENT_ID`, `GITHUB_APP_CLIENT_SECRET` — create GitHub App using prefilled link in README
- `DEFAULT_SANDBOX_PROVIDER` — set to `e2b` for simpler local setup (avoid Modal Python complexity)
- `E2B_API_KEY` — `e2b_REDACTED`

### 4. Install Dependencies
```bash
pnpm install
```

### 5. Start Infrastructure
```bash
docker compose up -d postgres redis
# Wait for healthy status
docker compose ps
```

### 6. Run Migrations
```bash
make db-migrate
# OR: pnpm --filter @proliferate/db db:migrate
```

### 7. Start Application
```bash
pnpm dev
# OR targeted:
make web        # web only (port 3000)
make gateway    # gateway only (port 8787)
```

### 8. Verify
- [ ] Web UI loads at http://localhost:3000
- [ ] Gateway responds at http://localhost:8787
- [ ] No TypeScript or build errors in console
- [ ] Can create account / org
- [ ] Can connect a GitHub repo (if GitHub App configured)

## If Blockers Arise

- **setup-env.sh fails on Windows**: Manually create `.env`, use `openssl rand -base64 32` for secrets
- **Docker not available**: Document and pivot to setting up Docker Desktop first
- **pnpm version mismatch**: Use `corepack enable && corepack prepare pnpm@8.15.1 --activate`
- **Port conflicts**: Check with `netstat -ano | findstr :3000` and kill conflicting process
- **Migration fails**: Check `DATABASE_URL` format — must be `postgresql://...?sslmode=disable` for local

Log all issues in `.claude/debugger/KNOWN_ISSUES.md` with severity + resolution.
